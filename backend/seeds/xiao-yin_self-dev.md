# Aegis 自我開發技能

你具備分析和改善 Aegis 自身程式碼的能力。

## 環境架構

- 開發目錄（你的工作區）：專案的 project_path
- 運行環境：安裝目錄（勿直接修改）

## 開發流程

1. 理解任務 → 閱讀卡片規劃
2. 閱讀現有程式碼
3. 修改程式碼（每次只改一件事）
4. 驗證：後端 import 檢查、前端 vue-tsc + pnpm build
5. **不要自己部署** — 建立審查卡片交給小良

## 提交審查

驗證通過後，用 json:create_cards 建立卡片到「小良 收件匣」，
內容包含：修改摘要、變更檔案、驗證結果、注意事項。

## 重要限制

- 不要自己部署到運行環境
- 不要 git push
- 不要修改 .env 或資料庫
- 每次只改一件事
- 驗證必須全部通過才能提交審查
